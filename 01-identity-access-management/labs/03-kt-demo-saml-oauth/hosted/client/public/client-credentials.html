On the wire — front channel vs back channel

    
Front channel = rides the browser's address bar (redirects). Back channel = a direct HTTPS call. IdP endpoints live under /realms/KT-idp/protocol/openid-connect.

    
    
      
  • Back channel — POST /token from our server (grant_type=client_credentials + client_secret): the service authenticates as itself.
    • 
      
  • Back channel — POST /token/introspect: the Resource Server asks the IdP "is this token alive?" before answering 200.
    • 
      
  • No front channel at all — no browser redirect, no login page, no consent. If a login screen ever appears in this flow, something is misconfigured.
